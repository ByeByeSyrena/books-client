# Component Patterns

- Make reusable components and split them into separate files when reuse or readability justifies it.
- Keep components small and focused.
- Do not create unnecessary abstractions for one-time logic.
- Use `.map` to render lists of the same repeated item.
- Keep API and business logic out of UI components.
- Use proper loading, error, and empty states.
- Avoid deeply nested JSX when it can be split into focused components.
- For forms, keep validation clear and close to schemas or contracts.
